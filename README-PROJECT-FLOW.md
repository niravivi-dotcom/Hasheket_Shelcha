# README — Avivi Solutions Pension Feedback Automation Pilot

## מהות ומטרת הפרויקט
**מטרה**: ליצור מערכת אוטומטית לעיבוד, זיהוי וטיפול בשגיאות בקבצי היזון חוזר פנסיוני. המערכת תנתח נתונים מה-API של "השקט שלך", תמפה שגיאות לפי קובץ מיפוי, תיצור טיוטות מייל אוטומטיות ב-Gmail, ותעדכן את מערכת המקור בתוצאות הטיפול.

## מהלך עבודה עיקרי (מעבר מפיילוט לייצור)
1. **קליטת נתונים מה-API של "השקט שלך"** (JSON): החלפת קבצי האקסל הידניים בשליפה אוטומטית.
2. **זיהוי רשומות באמצעות `MISPAR_MEZAHE_RESHUMA`**: שימוש ב-GUID כמפתח ראשי לסגירת מעגל.
3. **עיבוד ב"מוח" (Python Engine)**:
   - חישוב מונה (Counter) שבועי על בסיס `UpdateDate`.
   - החלת לוגיקת Override (דריסת אחריות) מול `LastPositive_CHODESH_MASKORET`.
4. **מיפוי קוד שגיאה לטיפול (ErrorCodeV4Id)**:
   - לפי טבלת "קודי שגיאה מקרים ותגובות" המעודכנת (`error_code_mapping_final`).
5. **יצירת טיוטות מייל אוטומטיות ב-Gmail**:
   - שימוש בפרטי קשר מתוך ה-API (`CustomerContactEmail`).
   - ה-Draft נוצר בתיבת מנהלת התיק הרלוונטית לפי שדה `CaseManagerEmail` מהרשומה.
   - n8n משתמש ב-Service Account עם Domain-wide Delegation (הוגדר 2026-03-14).
   - צירוף נספח אקסל ממוקד לכל מעסיק.
6. **דיווח חזרה (POST)**: עדכון המערכת של דוד במונה ובסטטוס הטיפול לכל רשומה.

## דגשים ואילוצים עיקריים
- **PII & Privacy**: הנתונים מעובדים בשרת ייעודי (Runner) המוגן ב-OAuth2.
- **שדות חסרים**: נדרשת השלמת `TikMislaka` ו-`OriginalFileName` ב-API לצורך הנספחים.
- **גמישות מיפוי**: המערכת תומכת בשמות עמודות משתנים (Hebrew/English) לצורך תאימות לאחור.
- **עדיפות לאוטומציה - אך נדרשת בקרת אנוש**: מנהלת תיק בודקת/מאשרת טיוטות טרם שליחה.

## אתגרים שזוהו עד כה
- טיפול בשגיאות רב-משתתפים (אותה שגיאה מרובת ת.ז/עובדים)
- קליטת קבצים במבנים שונים (עברית/אנגלית)
- שדה נושא/גוף דינמי למיילים (שיבוץ משתנים מהנתונים)
- סקירה ואישור טרם שליחה (לא הכל אוטומטי)
