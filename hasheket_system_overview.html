נקודות פתוחות לקראת שלב הטסטים

1. Google Drive — איזה חשבון? עידו

קובץ המיפוי (error_code_mapping_final.xlsx) צריך לשבת ב-Drive שאליו n8n יכול לגשת. צריך לאשר עם עידו איזה Google Workspace account מחובר ל-n8n ולוודא שהקובץ נמצא שם.

2. מיילים לטסט — כתובת אחת אחריות ניר

לצורך הטסט, כל הטיוטות יגיעו לתיבת מייל אחת (ולא לכל מנהלת תיק לפי רשומה). זה מוגדר אצלנו בלבד — env var ב-Render בשם TEST_EMAIL_OVERRIDE. לפני מעבר לייצור: מוחקים את ה-env var, וכל טיוטה הולכת לתיבה הנכונה אוטומטית. דוד לא צריך לשנות שום דבר.

3. שמירת Refresh Token אחרי רענון אחריות ניר

כרגע n8n שומר רק את ה-access_token החדש אחרי רענון, ה-refresh_token החדש נזרק. כשה-refresh_token הנוכחי יפוג (2027-03-01) — המערכת תפסיק לעבוד. צריך פתרון לשמירת ה-refresh_token המתחלף (n8n Variables / Supabase / דוד ייפיק זוג חדש ידנית).

4. הפעלת SetFeedbackStatus בפועל דוד

לצורך בדיקת end-to-end — לאשר עם דוד שה-endpoint כותב לבסיס הנתונים בסביבת הטסט (לפי המייל האחרון, הוא כן מומש, אך כדאי לאמת).

5. Chunking ב-SetFeedbackStatus אחריות ניר

דוד ביקש לא לשלוח 1,000 קריאות ברצף רציף. יש להוסיף delay בין batches ב-n8n (node-13). לתאם איתו את הגדלים והזמנים לפני ייצור.

6. Trigger שבועי — Cron אחריות ניר

כרגע ה-workflow מופעל ידנית. לפני ייצור: להחליף את node-01 (Manual Trigger) ל-Schedule Trigger בn8n. לקבוע יום ושעה (מומלץ: ראשון בלילה).
